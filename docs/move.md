# Move to final domains

## Goal

Three envs total:

| Env   | Web                   | CMS                     | D1/R2     | Type            |
| ----- | --------------------- | ----------------------- | --------- | --------------- |
| local | http://localhost:4321 | http://localhost:3000   | miniflare | local dev       |
| test  | test.codeuncode.com   | cms-test.codeuncode.com | remote    | remote pre-prod |
| live  | codeuncode.com        | cms.codeuncode.com      | remote    | remote prod     |

Two remotely deployed envs (test + live) + one local (miniflare). Each env is fully isolated — own worker, own D1, own R2, own secrets.

## Where we are now

Currently only `local` and `beta` are wired in wrangler. `beta` points at `cms-beta.codeuncode.com` / `beta.codeuncode.com`, with D1+R2 named `cu-core-staging`. No prod env is deployed yet — old-core still holds `cms.codeuncode.com` + `codeuncode.com` on the old `cu-core` / `cu-web` worker names.

Existing resources in this account:

- Workers: `cu-core-beta`, `cu-web-beta` (new-core), plus `cu-core`, `cu-web` (old-core).
- D1: `cu-core-staging` (used by beta).
- R2: `cu-core-staging` (used by beta).

## Target state

### Resource map

| Env   | CMS worker            | CMS route               | Web worker            | Web route           | D1                   | R2                   |
| ----- | --------------------- | ----------------------- | --------------------- | ------------------- | -------------------- | -------------------- |
| local | `next dev`            | localhost:3000          | `astro dev`           | localhost:4321      | miniflare            | miniflare            |
| test  | `codeuncode-cms-test` | cms-test.codeuncode.com | `codeuncode-web-test` | test.codeuncode.com | `codeuncode-test`    | `codeuncode-test`    |
| live  | `codeuncode-cms`      | cms.codeuncode.com      | `codeuncode-web`      | codeuncode.com      | `codeuncode-live`    | `codeuncode-live`    |

Naming note: adopting `codeuncode-*` for all new deployable resources means the live workers (`codeuncode-cms` / `codeuncode-web`) don't collide with old-core's `cu-core` / `cu-web`. New-core and old-core can coexist through Phase C; cutover is a DNS/route swap with no worker-name race.

All test resources are created fresh (new D1, new R2, new worker) rather than renaming `cu-core-staging`. Test data is disposable — we re-seed from `/seed` rather than exporting/importing beta data.

### Wrangler env layout

CMS (`apps/cms/wrangler.jsonc`):

- `env.local` — miniflare (unchanged).
- `env.test` — replaces `env.beta`.
- `env.live` — new.
- Top-level stays bindings-free (washi pattern: explicit `--env=<name>` on every deploy).

Web (`apps/web/wrangler.jsonc`) — currently has only a top-level config. We need to restructure into `env.test` + `env.live` and clear the top-level bindings.

### Secrets per env (set via `wrangler secret put --env=<env>`)

- `PAYLOAD_SECRET` — unique per env. Generate with `openssl rand -hex 32`.
- `SEED_SECRET` — unique per env.
- `RESEND_API_KEY` — shared across test + live (reuse old-core's key).
- Inventorying any other old-core secrets (Cron Triggers, Queues, DOs, service bindings, etc.) is deferred; revisit before live cutover.

## Execution plan

Three phases. Phase A is mostly a rename, Phase B provisions new live resources, Phase C is the cutover.

### Phase A — Stand up `test` env (replaces `beta`)

Fresh Cloudflare resources under the new naming. Old `cu-core-beta` / `cu-web-beta` workers + `cu-core-staging` D1/R2 get decommissioned at the end.

1. **DNS:** add `cms-test.codeuncode.com` and `test.codeuncode.com` CNAMEs (proxied) in the Cloudflare zone for `codeuncode.com`.
2. **Create D1:** `wrangler d1 create codeuncode-test` → record id.
3. **Create R2:** `wrangler r2 bucket create codeuncode-test`.
4. **CMS wrangler:**
   - Rename `env.beta` → `env.test`.
   - Set `name: "codeuncode-cms-test"`, route `cms-test.codeuncode.com`, `CMS_URL: https://cms-test.codeuncode.com`, `WEB_URL: https://test.codeuncode.com`.
   - D1 binding → `codeuncode-test` (id from step 2).
   - R2 binding → `codeuncode-test`.
5. **Web wrangler:**
   - Move current top-level into `env.test`.
   - Set `name: "codeuncode-web-test"`, route `test.codeuncode.com`, `PUBLIC_CMS_URL: https://cms-test.codeuncode.com`.
   - Clear top-level bindings (same washi pattern as CMS).
6. **Source refs:**
   - `apps/web/src/pages/preview/{about,index,work}.astro` CSP `frame-ancestors`: replace `https://cms-beta.codeuncode.com` with `https://cms-test.codeuncode.com` **and** `https://cms.codeuncode.com` (preview must work in both test + live).
   - `apps/cms/cloudflare-env.d.ts` — regenerated by `pnpm generate:types:cloudflare` after wrangler change.
   - `apps/cms/scripts/migration/README.md` — swap `cms-beta` → `cms-test`, `beta.` → `test.`, `cu-core-staging` → `codeuncode-test`.
   - `apps/cms/package.json` → `migration:import:beta` script: rename to `migration:import:test`, set `CLOUDFLARE_ENV=test`.
   - `apps/cms/scripts/migration/import-new.ts` header comment (line 22): `CLOUDFLARE_ENV=beta` → `test`.
   - `docs/content-migration.md` — swap beta→test naming.
7. **Secrets:** `wrangler secret put PAYLOAD_SECRET --env=test`, same for `SEED_SECRET` and `RESEND_API_KEY` (shared key, reuse the beta value).
8. **Run migrations:** `CLOUDFLARE_ENV=test pnpm --filter cu-core deploy:database` against the empty `codeuncode-test` D1.
9. **Deploy:** `pnpm --filter cu-core deploy --env=test`; `pnpm --filter cu-web deploy --env=test`.
10. **Bootstrap:** create admin user at `cms-test.codeuncode.com/admin`, hit `/seed?secret=$SEED_SECRET` to populate.
11. **Smoke test:** `cms-test.codeuncode.com/admin` logs in; `test.codeuncode.com` renders; live preview iframe works.
12. **Decommission beta:** `wrangler delete --name cu-core-beta`, `wrangler delete --name cu-web-beta`, delete `cu-core-staging` D1 + R2 via dashboard (after confirming test is healthy).

### Phase B — Provision `live` env (no cutover yet)

Stand up new-core's live workers on their `workers.dev` URLs **while old-core still serves codeuncode.com**. Lets us validate before DNS flip. Worker names don't collide with old-core, so this is a clean parallel deploy.

1. **Create D1:** `wrangler d1 create codeuncode-live` → record id.
2. **Create R2:** `wrangler r2 bucket create codeuncode-live`.
3. **CMS wrangler** — add `env.live`:
   - `name: "codeuncode-cms"`.
   - D1 binding → `codeuncode-live` (id from step 1).
   - R2 binding → `codeuncode-live`.
   - `CMS_URL: https://cms.codeuncode.com`, `WEB_URL: https://codeuncode.com`.
   - No `routes` yet (added in Phase C). `Users.auth.cookies.domain: .codeuncode.com` already in place in payload config.
4. **Web wrangler** — add `env.live`:
   - `name: "codeuncode-web"`, `PUBLIC_CMS_URL: https://cms.codeuncode.com`.
   - No `routes` yet.
5. **Secrets:** `wrangler secret put PAYLOAD_SECRET --env=live` (fresh), same for `SEED_SECRET` (fresh), `RESEND_API_KEY` (reuse shared key).
6. **Run migrations:** `CLOUDFLARE_ENV=live pnpm --filter cu-core deploy:database` → migrates empty `codeuncode-live` D1 up to current schema.
7. **Deploy apps** (no routes): `pnpm --filter cu-core deploy --env=live`; `pnpm --filter cu-web deploy --env=live`. They'll serve on `codeuncode-cms.<account>.workers.dev` / `codeuncode-web.<account>.workers.dev`.
8. **Admin bootstrap:** create first admin user at `codeuncode-cms.<account>.workers.dev/admin`. Confirm it writes into `codeuncode-live` D1, not beta.

### Phase C — Cutover to live domains

Content migration + DNS flip. This is the point of no return.

1. **Content migration** (per `docs/content-migration.md` — we already have scripts for exporting old-core D1 + R2 and importing into new core):
   - Export from old-core prod D1 + R2.
   - Import into `codeuncode-live` D1 + R2.
2. **Release domains from old-core:** remove `cms.codeuncode.com` + `codeuncode.com` routes from old-core's wrangler and redeploy old-core (or delete the routes from the CF dashboard). This is the one step that causes a brief serving gap — minimize it by having step 3 ready to go.
3. **Claim domains on new-core:**
   - Add `routes: [{ pattern: "cms.codeuncode.com", custom_domain: true }]` to CMS `env.live`.
   - Add `routes: [{ pattern: "codeuncode.com", custom_domain: true }]` to web `env.live`.
   - `pnpm --filter cu-core deploy --env=live`; `pnpm --filter cu-web deploy --env=live`.
4. **Smoke test:**
   - `cms.codeuncode.com/admin` — log in, content present.
   - `codeuncode.com` — public site renders with real content.
   - Live preview in prod: edit a Partner / Service / Project → iframe updates.
5. **Decommission:** once stable for N days, delete old-core workers, old `cu-core` prod D1 (after backup), old `cu-core` prod R2 (after backup).

## Decisions

- **Test resources:** fresh — new `codeuncode-test` D1 + R2 + worker. Old `cu-core-staging` + `cu-core-beta` get deleted at the end of Phase A.
- **Resend key:** shared across test + live. Reuse old-core's key.
- **DNS zone:** `codeuncode.com` is in this same Cloudflare account — no cross-account coordination needed.
- **Env key name:** `live` (not `production`).
- **Worker names:** `codeuncode-*` everywhere for new-core. No collision with old-core's `cu-core` / `cu-web`, so Phase B can deploy in parallel with old-core.
- **Old-core prod inventory:** deferred — revisit before Phase C cutover.
- **`docs/plan.md`:** move.md now owns the domain-move sub-plan (was plan.md's Phase 6). plan.md still covers Phases 0–5 of the rebuild; don't delete it. Update its Phase 6 section to point here.
